# CLAUDE.md

Working notes for AI assistants editing this repo. Read this before making changes.

## Stack & environment

- React 18 + Vite 5 + Framer Motion 11. Plain CSS (no framework), no router library,
  no TypeScript, no test suite.
- **No local Node/npm in this environment.** Don't run `npm install`/`npm run build`/the
  dev server here — they aren't available. Verify changes by reading the code, not by
  running it. Builds and deploys happen in CI (GitHub Actions, Node 24).

## Architecture

- **Entry:** `index.html` → `src/main.jsx` → `src/App.jsx`.
- **Routing** is query-param based in `App.jsx`: `getPage()` reads `?page` (`home` |
  `strava` | `films`) and `App` renders the matching page. `navigate(p)` pushes
  history (`/` for home, `/?page=<p>` otherwise) and scrolls to top; a `popstate`
  listener keeps back/forward working. There is no router library.
- **Content is data-driven.** `src/data/content.js` is the single source of truth for
  all copy, links, and lists. Components map over its exports — edit data there, do not
  hardcode strings in components.
- **Pages** (`src/pages/`): `Home` composes the home-page sections; `Strava` and `Films`
  are standalone. **Components** (`src/components/`): home-page sections plus shared
  `Navbar`, `Footer`, `Reveal`, and `icons`.

## Conventions

- One component = `Name.jsx` + `Name.css` in the same folder. Global styles and design
  tokens live in `src/styles/global.css`.
- Class names are BEM-ish: `block__element--modifier`.
- **Design tokens are CSS variables** in `global.css` `:root`, overridden under
  `:root[data-theme="dark"]`. Never hardcode a color/spacing/easing — use or add a token.
- Mono font (JetBrains Mono) for headings, labels, and structural text; sans (IBM Plex
  Sans) for body copy.
- Wrap content in `<Reveal>` for the scroll-into-view animation. It re-animates every
  time it enters the viewport (`once: false`) — this is intentional.
- Sections are `.section` (light) or `.section--dark` (warm gray in light mode,
  near-black in dark mode, with a dot-grid texture). Cards use the `--card-*` tokens so
  they render correctly on both.

## Theme

`data-theme="dark"` on `<html>` drives dark mode. An inline script in `index.html` sets
it from `localStorage` (falling back to `prefers-color-scheme`) **before React mounts**,
preventing a flash. `App.jsx` owns the toggle and persists the choice to `localStorage`.

## Data shapes

- **Films** (`films.topMovies`, `films.watchlist` in `content.js`) both render through the
  shared `MovieCard`: `{ title, year, director, rating /* 0–10 */, poster /* url */, note? }`.
- **Strava** page reads `public/strava-data.json`:
  `{ athlete, stats.ytd_run_totals, activities[], photos[], updated_at }`. Missing/empty
  data falls back to a built-in empty state. This file is generated by `strava.yml` —
  don't hand-edit it except for local testing.

## Common changes

- **Edit copy / projects / skills / films:** edit `src/data/content.js`.
- **Restyle globally:** edit the token values in `global.css` `:root`.
- **Add a home-page section:** create `Name.jsx` + `Name.css` in `components/`, add its
  content to `content.js`, render it in `pages/Home.jsx`, and add a `navLinks` entry with
  `type:'anchor'` whose `href` matches the section's `id`.
- **Add a standalone page:** add a `navLinks` entry `{ label, page, type:'page' }` in
  `content.js`, create `pages/Name.jsx` + `Name.css`, and add a render branch in `App.jsx`.

## Versioning

- **`package.json` `version` is the single source of truth** (SemVer `MAJOR.MINOR.PATCH`).
  The site is pre-1.0. Don't duplicate the version anywhere else.
- Vite injects it at build time as the `__APP_VERSION__` constant (`vite.config.js`
  `define`), which `Footer.jsx` reads and renders. To show the version somewhere new,
  reference `__APP_VERSION__` — never hardcode a version string.
- Bump rules: **PATCH** for content tweaks/fixes, **MINOR** for a new section/page/feature,
  **MAJOR** for a redesign or breaking structural change.
- **Release process** (run by the repo owner, not automatically): bump `version` in
  `package.json`, add a dated entry to `CHANGELOG.md`, commit as
  `chore(release): vX.Y.Z`, then tag it: `git tag -a vX.Y.Z -m "vX.Y.Z"`. Everyday
  work uses normal Conventional Commits (`feat:`, `fix:`, `chore:`); the version only
  changes on a release commit.

## Invariants / gotchas

- A `navLinks` entry with `type:'page'` needs a matching render branch in `App.jsx`;
  `type:'anchor'` needs its `href` to match a section `id` on the home page.
- In `projects`, `link: '#'` hides the card's "View" button.
- Deploy: pushing to `main` triggers `deploy.yml`. Commits made by `strava.yml` use
  `GITHUB_TOKEN`, which does **not** auto-trigger other workflows, so that workflow calls
  `gh workflow run deploy.yml` explicitly. Keep that step if you touch `strava.yml`.
