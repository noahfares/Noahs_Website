# Noah's Website

A personal site — clean, Apple-inspired, single-page. Built with **React** + **Vite**,
animated with **Framer Motion**, and deployed to **GitHub Pages** at
[noahfares.com](https://noahfares.com).

## What it is

- A home page (hero, about, experience, skills, contact) plus two standalone pages:
  **Strava** (auto-synced running/cycling activity) and **Films** (favorite movies and a watchlist).
- Nearly all page copy lives in one data file — `src/data/content.js` — so most edits
  don't touch component code.
- Light/dark theme with no flash on load; respects `prefers-reduced-motion`.

## Project structure

```
index.html              # Page shell; sets the theme before React mounts
vite.config.js          # Dev server config (port 5173)
public/
  CNAME                 # Custom domain (noahfares.com)
  favicon.svg
  strava-data.json      # Generated by the Strava workflow; read by the Strava page
src/
  main.jsx              # React entry point
  App.jsx               # Theme state + query-param page routing
  data/content.js       # All site copy, links, and lists — edit here
  styles/global.css     # Design tokens (CSS variables) + resets
  components/           # Home-page sections + shared pieces (Navbar, Footer, Reveal, icons)
  pages/                # Home, Strava, Films
.github/workflows/
  deploy.yml            # Build + deploy to GitHub Pages on push to main
  strava.yml            # Refresh strava-data.json on a schedule
```

Each component owns a `.jsx` and a matching `.css` in the same folder. Shared/global
styles live in `src/styles/global.css`.

## How it works

- **Routing** uses a query param (`/?page=strava`), not a router library. `App.jsx`
  reads `?page` and renders Home, Strava, or Films, updating browser history on nav.
- **Content is data-driven**: components map over the arrays/objects exported from
  `content.js`. Add a project, skill, or film by editing that file.
- **Strava** data is fetched by `strava.yml` (a scheduled GitHub Action), written to
  `public/strava-data.json`, committed, and picked up on the next deploy.
- **Deploy is automatic**: every push to `main` runs `deploy.yml`, which builds with
  Vite and publishes `dist/` to GitHub Pages.

## Local development

Requires Node 18+ (CI uses Node 24). From the project root:

```bash
npm install      # first time only
npm run dev      # http://localhost:5173, with hot reload
npm run build    # production build → dist/
npm run preview  # serve the production build locally
```

You rarely need to build by hand — pushing to `main` deploys the site.
